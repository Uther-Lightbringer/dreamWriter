📖 章节故事概括功能说明

✨ 新增功能
已为章节添加故事概括功能，用户可以在创建章节时填写该章节的故事概括， 在小说详情页的章节列表中会展示每个章节的故事概括。
🔧 后端修改
1. 修改Chapter实体类，添加storySummary字段：
@Column(name = "story_summary", columnDefinition = "TEXT") private String storySummary;
2. 修改NovelController的createChapter接口，接收storySummary参数：
@PostMapping("/novels/{novelId}/chapters") public ResponseEntity<?> createChapter( @PathVariable Long novelId, @RequestBody Map<String, String> request) { String storySummary = request.get("storySummary"); // ... 其他逻辑 // 设置故事概括 if (storySummary != null && !storySummary.trim().isEmpty()) { chapter.setStorySummary(storySummary.trim()); } return ResponseEntity.ok(chapter); }
🎨 前端修改
1. 修改小说详情页模板，在章节列表中显示故事概括：
<!-- 故事概括 --> <div th:if="${chapterWithTags.chapter.storySummary}" class="story-summary"> <div class="summary-label">📋 故事概括:</div> <div class="summary-content" th:text="${chapterWithTags.chapter.storySummary}"></div> </div>
2. 修改添加章节页面，增加故事概括输入框并使用AJAX提交：
<div class="form-group"> <label for="storySummary">故事概括：</label> <textarea id="storySummary" name="storySummary" placeholder="请输入本章节的故事概括（可选）" rows="4"></textarea> <small style="color: #7f8c8d; font-size: 12px;"> 简要描述本章节的主要情节和发展 </small> </div>

第3章

✅ 功能特点
• 可选填写：故事概括为可选字段，不影响原有功能
• 清晰展示：在章节列表中以醒目的样式展示故事概括
• 响应式设计：适配各种屏幕尺寸
• 无缝集成：与现有章节标签功能完美配合
• 易于使用：界面简洁直观，操作便捷
