🎭 角色卡版本号功能说明

📋 功能概述

角色卡系统现在支持独立的版本号管理，分别追踪 AI 绘画提示词和图片的生成次数。

📝 提示词版本号 (promptVersion)

每次重新生成 AI 绘画提示词时递增

触发操作：点击"重新生成 AI 绘画提示词"按钮

v1 → v2 → v3

🎨 图片版本号 (imageVersion)

每次使用提示词生成图片时递增

触发操作：点击"生成角色图片"按钮

v1 → v2 → v3

📊 版本号规则

字段名 初始值 递增时机 说明
promptVersion 0 重新生成 AI 绘画提示词 记录提示词的生成次数
imageVersion 0 生成角色图片 记录图片的生成次数
示例场景：
1. 第一次生成提示词 → promptVersion: 1
2. 第一次生成图片 → imageVersion: 1
3. 第二次重新生成提示词 → promptVersion: 2
4. 第二次重新生成图片 → imageVersion: 2
5. 第三次重新生成提示词 → promptVersion: 3

🔌 API 接口

1. 重新生成 AI 绘画提示词

请求：
POST /api/novels/{novelId}/character-cards/{characterId}/generate-prompt

返回：
{
  "success": true,
  "message": "AI 绘画提示词生成成功",
  "data": {
    "id": 1,
    "name": "张三",
    "promptVersion": 3, ← 提示词版本号
    "imageVersion": 1, ← 图片版本号
    ...
  }
}

2. 生成角色图片

请求：
POST /api/novels/{novelId}/character-cards/{characterId}/generate-image

返回：
{
  "success": true,
  "message": "角色图片生成成功",
  "data": {
  "id": 1,
  "name": "张三",
  "promptVersion": 3, ← 提示词版本号不变
  "imageVersion": 2, ← 图片版本号 +1
  ...
  }
}

💾 数据库变更

执行以下 SQL 脚本添加版本号字段：

-- 添加提示词版本号字段
ALTER TABLE character_cards ADD COLUMN prompt_version INTEGER DEFAULT 0;

-- 添加图片版本号字段
ALTER TABLE character_cards ADD COLUMN image_version INTEGER DEFAULT 0;

SQL 脚本位置：add_character_card_version.sql

💡 使用场景

场景 1：多次调整提示词

你想要不同的提示词风格，每次重新生成都会计入版本号：

  • 第 1 次：生成标准提示词 → promptVersion: 1
  • 第 2 次：生成详细版提示词 → promptVersion: 2
  • 第 3 次：生成简洁版提示词 → promptVersion: 3

场景 2：多次尝试生成图片

同一份提示词，想要多张不同的图片：

  • 第 1 次生成 → imageVersion: 1
  • 第 2 次生成 → imageVersion: 2
  • 第 3 次生成 → imageVersion: 3

场景 3：独立管理

提示词和图片可以独立更新：

  • 更新了 3 次提示词 → promptVersion: 3
  • 只生成了 1 次图片 → imageVersion: 1
  • 两个版本号互不影响！
